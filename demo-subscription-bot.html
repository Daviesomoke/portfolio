Telegram Subscription
Manager Bot

Automates the full lifecycle of paid Telegram channel access from payment confirmation through to expiry and removal. Built for creators, communities, and businesses that sell gated content without wanting to manage subscribers manually.

Python python-telegram-bot SQLite M-Pesa API Webhook Cron Jobs

What it handles

  • User sends /subscribe bot responds with plan options and payment instructions
  • M-Pesa STK push is triggered; bot listens for the IPN callback to confirm payment
  • On confirmation, user is added to the private channel via invite link with expiry tracked in the database
  • Renewal reminders sent at 3 days and 1 day before expiry via scheduled jobs
  • Expired subscribers are removed automatically no manual intervention required
  • Admin commands: view active subs, manually extend, revoke, or export a CSV report
  • All state is persisted in SQLite portable and zero external dependencies

Execution flow

1
User sends /subscribe Bot presents plan tiers and pricing inline
2
M-Pesa STK push initiated User receives payment prompt on their phone
3
IPN callback received Bot verifies transaction ID and amount
4
Access granted User is sent a one time invite link to the channel
5
Expiry handled automatically Reminders sent, then user removed on expiry date
